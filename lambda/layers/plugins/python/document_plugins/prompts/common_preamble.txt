GENERAL NORMALIZATION RULES:

1. **Interest Rates and Percentages**:
   - Convert ALL percentage values to decimal format (e.g., "8.5%" -> 0.085, "4.25%" -> 0.0425)
   - Handle text formats ("Five and a half percent" -> 0.055)
   - For basis points: 50 bps = 0.005, 150 bps = 0.015

2. **Currency / Dollar Amounts**:
   - Convert to numeric format without symbols (e.g., "$250,000.00" -> 250000.00)
   - Handle text formats ("Two hundred fifty thousand" -> 250000.00)
   - Remove commas and currency symbols

3. **Names**:
   - Preserve names exactly as they appear in the source document
   - For ALL-CAPS names, convert to Title Case ("SMITH, JOHN A" -> "John A Smith")
   - Preserve full legal entity names with suffixes (LLC, Inc., Corp.)

4. **Dates**:
   - Convert ALL dates to ISO 8601 format: YYYY-MM-DD
   - Handle various input formats ("01/15/2024", "January 15, 2024", "1/15/24", etc.)
   - If only month and year are given, use first of month (e.g., "March 2024" -> "2024-03-01")

5. **Addresses**:
   - Preserve as structured objects when the schema calls for it
   - Normalize state abbreviations to 2-letter codes (e.g., "Texas" -> "TX")
   - Preserve zip codes as strings (leading zeros matter)

6. **Missing / Unclear Data**:
   - If data cannot be confidently extracted, use null
   - NEVER hallucinate or guess missing values
   - Do NOT use empty strings "" -- use null instead
   - Add a note in the validationNotes array for any uncertain or missing critical fields

7. **Boolean Fields**:
   - Return true or false (JSON boolean), not strings
   - If indeterminate, return null with a validation note
